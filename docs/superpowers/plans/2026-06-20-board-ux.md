# Board UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add activity-bar Excalidraw + Mindmap board-create buttons (Export moved before Gear) and make the jsMind board follow the active MDflow theme.

**Architecture:** Two new activity-bar buttons create untitled board tabs (`Untitled.excalidraw` / `Untitled.mind`) that ride the existing untitled-save flow. jsMind theming is CSS keyed off the app theme plus a per-theme `line_color` option set on mount.

**Tech Stack:** TypeScript (no framework), jsMind, Tauri 2. Design spec: `docs/superpowers/specs/2026-06-20-board-ux-design.md`.

## Global Constraints

- Clean-room: no Kaelio code/CSS.
- Activity-bar order (exact): Explorer · Search · AI · Excalidraw · Mindmap · Export · Gear.
- New board content: empty Excalidraw scene; mindmap root topic exactly "Central Idea" (from the existing `parseMindmap("")`/`parseExcalidrawDocument("")`).
- jsMind theme follows the app theme (light/dark); no fixed jsMind theme name.
- Small focused changes, frequent commits.

- Modify: `src/activitybar.ts` (wire the two buttons)
- Modify: `src/main.ts` (the `onNewBoard` callback)

**Interfaces:**
- Produces: `initActivityBar(..., onExport, onNewBoard)` where `onNewBoard(kind: "excalidraw" | "mind")` opens a new untitled board.

- [ ] **Step 1: Markup + reorder**

In `index.html`, in `#activitybar`, after `#ab-ai` add:
```html
<button class="ab-btn ab-excalidraw" id="ab-excalidraw" title="New Excalidraw" type="button" aria-label="New Excalidraw board"></button>
<button class="ab-btn ab-mindmap" id="ab-mindmap" title="New Mindmap" type="button" aria-label="New Mindmap board"></button>
```
Then move the existing `#ab-export` button so it sits immediately **before** `#ab-settings`. Final order: ab-explorer, ab-search, ab-ai, ab-excalidraw, ab-mindmap, ab-export, ab-settings.

- [ ] **Step 2: Glyphs**

In `src/glyphs.ts`, add two themeable 16px line glyphs (stroke `currentColor`) following
the existing entries: `excalidraw` (a simple pencil-on-rectangle mark) and `mindmap` (a
central node with two branching child nodes). Keep them visually distinct from `read`/`ai`.

- [ ] **Step 3: Wire activitybar**

In `src/activitybar.ts`, add an `onNewBoard: (kind: "excalidraw" | "mind") => void = () => {}`
parameter; set the two new buttons' icons from `glyphs.excalidraw`/`glyphs.mindmap`; add
click listeners calling `onNewBoard("excalidraw")` / `onNewBoard("mind")`.

- [ ] **Step 4: Implement `newBoard` in main.ts**

```ts
function newBoard(kind: "excalidraw" | "mind"): void {
  const name = kind === "excalidraw" ? "Untitled.excalidraw" : "Untitled.mind";
  openInWindow(getState().activeWindowId, { path: null, name, text: "" });
}
```
Pass `newBoard` as the new `onNewBoard` argument to `initActivityBar(...)`. (Because the
tab name carries the extension, `isExcalidrawFile`/`isMindmapFile` route it to the board
pane and the empty text yields each format's default; first ⌘S prompts for a path via the
existing Save-As flow.)

- [ ] **Step 5: Build + manual check**

Run: `npm run build`
Expected: clean. (Manual: clicking each button opens a new board; order is correct.)

- [ ] **Step 6: Commit**

```bash
git add index.html src/glyphs.ts src/activitybar.ts src/main.ts
git commit -m "feat(board): activity-bar new-Excalidraw/new-Mindmap buttons + reorder"
```

### Task 2: jsMind theming follows the app theme

**Files:**
- Modify: `src/styles.css` (jsMind board theme CSS)
- Modify: `src/mindmapview.ts` (per-theme line color on mount)

**Interfaces:**
- Consumes: `mountMindmapBoard` (existing).

- [ ] **Step 1: Board CSS keyed to the app theme**

In `src/styles.css`, add rules for jsMind's DOM that derive from the app theme variables,
scoped under the board pane, e.g.:
```css
.window-document-board jmnode {
  background: var(--bg-elev); color: var(--text);
  border: 1px solid var(--border); border-radius: 8px;
}
.window-document-board jmnode.root {
  background: var(--accent); color: #1b1b1b;
}
.window-document-board jmnode:hover { border-color: var(--accent); }
.window-document-board jmnode.selected { box-shadow: 0 0 0 2px var(--accent); }
.window-document-board jmexpander { color: var(--muted); }
.window-document-board .jsmind-inner { background: var(--bg-preview); }
```
Because the variables already change with theme, light themes get a light board and dark
themes a dark board automatically.

- [ ] **Step 2: Connector line color per theme**

In `src/mindmapview.ts`, when constructing jsMind, set the connector line color to match
the theme by reading a CSS variable from the document, e.g.:
```ts
const styles = getComputedStyle(document.documentElement);
const lineColor = (styles.getPropertyValue("--border").trim() || "#888");
const jm = new JsMind({ container: host, editable: true, mode: "full",
  view: { line_color: lineColor } });
```
(Confirm the exact option path during implementation — jsMind 0.9 reads
`options.view.line_color`. If a remount is needed on theme change, that already happens
because the board re-renders; no extra wiring required for M11's current flow.)

- [ ] **Step 3: Build + manual check**

Run: `npm run build`
Expected: clean. (Manual: open a `.mind` in a dark theme → legible dark board; switch to
a light theme → light board.)

- [ ] **Step 4: Commit**

```bash
git add src/styles.css src/mindmapview.ts
git commit -m "feat(board): jsMind board follows the app theme"
```
